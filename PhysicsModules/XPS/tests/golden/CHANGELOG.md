# Golden baseline changelog

Every (re)capture of files under `expected/` gets an entry here: date, cases,
reason, author. CI policy (Phase 1): expected files must not change without a
matching entry.

## 2026-07-04 — all baselines rebaselined: Phase 4.2 RNG-sequence change

- **Files:** `expected/*` (9 CLI cases), `expected_analysis/*` (5)
- **Reason:** the Shirley background functions constructed a throwaway
  `peak` on every fitness evaluation whose 8 unused `np.random.choice`
  draws consumed the main RNG sequence. They now use the RNG-free
  container mode (drawn values were never used — the shape functions
  receive all parameters explicitly), which is required for `--workers`
  parallel evaluation to follow the serial trajectory and also removes
  per-evaluation range construction. This legitimately shifts every
  subsequent RNG draw → new trajectories (design.md's re-baseline
  protocol for RNG-sequence changes). Curve math is unchanged
  (component references untouched); `--workers 4` verified byte-identical
  to serial on 5 cases plus a population-2000 run.
- **Sign-off:** Andy Lau (directed 2026-07-04: "perform all of the
  remaining candidates as shown in performance.md")

## 2026-07-04 — analysis goldens rebaselined: unified curve math (Phase 3d)

- **Files:** `expected_analysis/*.txt` (all 5)
- **Reason:** deliberate behavior change completing the fork unification.
  `gui/xps_analysis2` now rebuilds the best individual with the package's
  `Individual` (analysis container mode) and evaluates it with the same
  `getFit` the GA's fitness used — so displayed/analyzed curves are by
  construction the curves the GA optimized, replacing the gui fork's
  diverged shape formulas. Observed effect: differences at float-noise to
  ~1e-6 relative level (e.g. `voigt_baseline_svsc` loss now matches the
  GA's value to the last digit; `gaussian_linear` shifted ~2e-7 from the
  Gaussian construction unification). Peak components remain composed on
  top of the total background, as the analysis's area computation expects.
  CLI goldens unaffected (bit-identical, 9/9). Deleted `gui/xps_fit.py`
  and `gui/xps_individual.py`.
- **Author:** Andy Lau

## 2026-07-04 — analysis goldens rebaselined: unified loss (Phase 3)

- **Files:** `expected_analysis/*.txt` (all 5)
- **Reason:** deliberate behavior change. `gui/xps_analysis2.fitness` now
  calls the canonical `xps_neo.loss.compute_loss` (the function the GA
  optimizes) with `apply_penalties=False` — the GA's final-phase
  definition — instead of its diverged private copy (N=30 window,
  penalties always on). Effects: (1) reported loss values now match the
  GA's logged "Best fit" scale; (2) in 2 of 5 cases (`gaussian_linear`,
  `two_peak_limited_correlated`) the analysis now selects a different
  best-parameter row — verified to be **exactly the row the GA itself
  reported as its global best**, which the old metric had not picked.
  CLI goldens unaffected (bit-identical). Decision log:
  `docs/fork-divergence.md`.
- **Author:** Andy Lau
